# CI Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions workflow that runs the Bun test suite and a compiled binary E2E test on every push and PR.

**Architecture:** Single workflow file with two sequential jobs — `test` (typecheck + unit tests + lint in dev mode) and `binary` (compile binary, copy WASM sidecars, run build→query→explain-edges against the fixture project). The `binary` job depends on `test` so it only runs when the suite is green.

**Tech Stack:** GitHub Actions, `oven-sh/setup-bun@v2`, `actions/checkout@v4`, Bun 1.x, bash.

### Task 1: Create the CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: ["**"]
  pull_request:
    branches: [main]

jobs:
  test:
    name: Test (dev mode)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Typecheck
        run: bun run typecheck

      - name: Test
        run: bun test

      - name: Lint
        run: bunx @biomejs/biome check .

  binary:
    name: Binary E2E
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Compile binary
        run: bun run build

      - name: Copy WASM sidecars
        run: |
          cp node_modules/web-tree-sitter/web-tree-sitter.wasm dist/
          cp node_modules/tree-sitter-typescript/tree-sitter-typescript.wasm dist/

      - name: E2E — build fixture
        run: ./dist/tsgraph build ./fixtures/simple-ts

      - name: E2E — query
        run: |
          set -euo pipefail
          ./dist/tsgraph query greet --db ./fixtures/simple-ts/.tsgraph.db > /tmp/query-out.json
          grep -q '"name"' /tmp/query-out.json
          grep -q 'greet' /tmp/query-out.json

      - name: E2E — explain-edges
        run: |
          set -euo pipefail
          ./dist/tsgraph explain-edges > /tmp/edges-out.json
          grep -q 'edge_kinds' /tmp/edges-out.json

      - name: Cleanup fixture DB
        if: always()
        run: rm -f fixtures/simple-ts/.tsgraph.db fixtures/simple-ts/.tsgraph.db-wal fixtures/simple-ts/.tsgraph.db-shm
```

Expected: both `test` and `binary` jobs green (✓). This may take 2–4 minutes.

- [ ] **Step 3: If `test` job fails — diagnose**

```bash
gh run view --log-failed
```

Common causes:
- `bun install --frozen-lockfile` fails → run `bun install` locally and commit updated `bun.lock`
- `biome check` fails → run `bunx @biomejs/biome check --write .` locally, commit fixes
- `typecheck` fails → run `bun run typecheck` locally, fix errors

- [ ] **Step 4: If `binary` job fails — diagnose**

Most likely causes:
- WASM path wrong → verify `node_modules/web-tree-sitter/web-tree-sitter.wasm` exists locally: `ls node_modules/web-tree-sitter/*.wasm`
- Binary not executable → the `bun run build` script should produce an executable; check `ls -la dist/tsgraph`
- `tsgraph build` fails with WASM error → check that both WASM files were copied to `dist/` before the binary ran
- `grep` fails → inspect `/tmp/query-out.json` with `cat` step or check `tsgraph query` output manually

If the binary job fails due to WASM loading, add a debug step before E2E:
```yaml
- name: Debug — list dist/
  run: ls -la dist/
```
